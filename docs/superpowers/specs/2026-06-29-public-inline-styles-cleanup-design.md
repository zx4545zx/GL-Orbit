# Design: Public Inline Styles Cleanup

Date: 2026-06-29

## Goal

Reduce SEO checker "Inline Styles" warnings on public GL-Orbit pages by removing inline `style=` attributes from public pages and shared public-facing components where practical, without changing the visual design or behavior.

## Current Context

A source scan found 41 `style=` matches. One global inline style appears in `src/app.html`, so it is emitted on every public page. Public-facing route/component matches are concentrated in:

- `src/app.html`
- `src/routes/(app)/+page.svelte`
- `src/routes/(app)/artists/[id]/+page.svelte`
- `src/routes/(app)/profile/+page.svelte`
- `src/routes/(app)/countdown/+page.svelte`
- `src/lib/components/ShareButton.svelte`
- `src/lib/components/BottomNav.svelte`
- `src/lib/components/chat/ChatApp.svelte`

There are also inline styles in generated SVG markup for `src/routes/og-image/+server.ts` and admin-only UI. These are out of scope unless they affect public HTML page audits.

## Chosen Approach

Clean up public pages and shared public-facing components only. Leave admin-only pages unchanged for this pass.

This targets the user-selected scope: all public pages, while avoiding unrelated admin UI changes.

## Design Principles

- Preserve current visual design and behavior.
- Replace static inline styles with Tailwind utility classes when available.
- Replace repeated custom values with named CSS classes in component `<style>` blocks or existing global CSS utilities.
- Keep dynamic values only when they are truly data-driven and cannot be represented with a small finite class map.
- Do not introduce new dependencies.
- Do not change route data loading, database access, authentication, or SEO endpoint behavior.

## Cleanup Patterns

### App shell wrapper

Replace the SvelteKit body wrapper inline style:

```html
<div style="display: contents">%sveltekit.body%</div>
```

with the equivalent Tailwind class:

```html
<div class="contents">%sveltekit.body%</div>
```

### Static performance hints

Replace inline declarations such as:

```html
style="will-change: transform"
```

with classes such as:

```html
class="will-change-transform"
```

Tailwind already supports `will-change-transform`.

### Static animation delays and durations

Replace fixed inline values such as:

```html
style="animation-delay:-6s; animation-duration:14s;"
```

with named local classes, for example:

```css
.orbit-delay-lavender {
	animation-delay: -6s;
	animation-duration: 14s;
}
```

### Dynamic loop animation delays

Where loop indexes create a small finite set of animation delays, use a helper function or class map to select a class.

Example concept:

```ts
const staggerClasses = ['stagger-0', 'stagger-1', 'stagger-2'];
function staggerClass(index: number): string {
	return staggerClasses[index] ?? staggerClasses[staggerClasses.length - 1];
}
```

Then define the matching classes in a local `<style>` block.

### Brand colors and icon sizes

In `ShareButton.svelte`, replace inline brand background and SVG size styles with named classes such as:

- `.share-brand-line`
- `.share-brand-facebook`
- `.share-icon-line`
- `.share-icon-facebook`
- `.share-icon-x`

### Safe-area spacing

Replace inline safe-area padding with named classes in the component style block, preserving the same `env(safe-area-inset-*)` expressions.

### Dynamic widths

Avoid changing admin dynamic progress widths in this pass. For public profile toggles, prefer class-based state mapping when there are only two possible states.

## Out of Scope

- Admin-only inline styles.
- `src/routes/og-image/+server.ts` SVG string styles, unless a public page audit specifically flags the image endpoint SVG itself.
- Large visual redesigns.
- Tailwind configuration changes unless a class cannot be expressed locally.
- Database/schema/auth changes.

## Verification

After implementation:

1. Run a targeted source scan for public inline styles:

```bash
rg -n 'style=' src/app.html 'src/routes/(app)' src/lib/components
```

Expected: no remaining `style=` matches in public page/component scope, or only documented exceptions that are not emitted into audited public HTML.

2. Run project validation:

```bash
npm run check
```

Expected: command exits with status 0. Existing warnings may remain if unrelated to this change.

3. Optionally run the dev server and inspect key pages:

- `/`
- `/artists/<id>` when local data is available
- `/countdown`
- `/profile` when authenticated
- `/chat` or any page using chat UI

Expected: visuals and interactions are unchanged.

## Risks

- Animation timing may subtly change if delay classes are mapped incorrectly.
- Safe-area spacing may regress on mobile if CSS expressions are copied incorrectly.
- Some inline styles may be generated by framework internals or third-party snippets and cannot be removed from source-level Svelte code.

Mitigation: keep changes mechanical, local, and verified with source scan plus `npm run check`.
