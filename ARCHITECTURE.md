# GL-Orbit Architecture

> Central hub and broadcast schedule for GL (Girls' Love) series worldwide. A SvelteKit web application with timezone-aware scheduling, member system, and admin panel.

---

## Tech Stack

| Layer | Technology | Version |
|-------|-----------|---------|
| Framework | SvelteKit | 2.x |
| UI | Svelte 5 (Runes) | ^5.25 |
| Language | TypeScript | ^5.8 |
| Styling | Tailwind CSS | ^4.0 |
| Build Tool | Vite | ^6.2 |
| Database | PostgreSQL (Neon) | serverless |
| ORM | Drizzle ORM | ^0.43 |
| DB Driver | `@neondatabase/serverless` | ^1.0 |
| Auth | Custom JWT (jose + bcryptjs) | jose ^6.2, bcryptjs ^3.0 |

## Core Components

### Authentication Layer (`src/lib/server/auth/`)

| Module | Responsibility |
|--------|---------------|
| `session.ts` | JWT create/validate/destroy using `jose` (HS256, 30-day expiry). Stores SHA-256 token hash in DB, not raw token. Clock tolerance 60s. |
| `password.ts` | bcrypt hash/verify with 12 rounds. |
| `user.ts` | DB queries: `getUserByEmail`, `getUserByUsername`, `getUserByIdentifier`, `createUser`. |

### Database Layer (`src/lib/server/db/`)

| Module | Responsibility |
|--------|---------------|
| `schema.ts` | Drizzle ORM schema: 10 tables with UUID v4 PKs, soft delete (`deleted_at`), audit fields (`created_at`, `updated_at`), enums for roles and status. |
| `index.ts` | Lazy initialization + Proxy pattern for `@neondatabase/serverless` HTTP connection. **Must use `getDb()` in server files** вЂ” the `db` proxy does not work in SSR context. |

### Pending Shells (`src/lib/components/*PendingShell.svelte`)

Skeleton UI components that display during client-side navigation to reduce perceived latency.

| Component | Purpose |
|-----------|---------|
| `PublicBaseShell.svelte` | Base layout for public pages (glassmorphism container, animations) |
| `AdminBaseShell.svelte` | Base layout for admin pages (header, breadcrumbs, table skeleton) |
| `SeriesPendingShell.svelte` | Series listing skeleton |
| `SeriesDetailPendingShell.svelte` | Series detail skeleton |
| `CalendarPendingShell.svelte` | Calendar view skeleton |
| `ProfilePendingShell.svelte` | User profile skeleton |
| `Admin*PendingShell.svelte` | 9 admin page shells (series, artists, studios, platforms, episodes, schedules, artist-socials, series-artists, episode-schedules) |

**Pattern:** Page shells compose base shells using Svelte 5 snippets (`{@render content()}`). Navigation detection uses `$derived` with `$app/state` (not `$app/stores`). Shells are SSR-safe (never render server-side) and accessible (`aria-busy`, `aria-live`, respects `prefers-reduced-motion`).

### Route Architecture

| Route Group | Purpose | Auth |
|-------------|---------|------|
| `(app)/` | Public-facing pages | None required |
| `profile/` | User profile management | USER or ADMIN |
| `admin/` | Full CRUD admin panel | ADMIN only |
| `api/` | JSON REST endpoints | Varies by endpoint |
| `logout/` | Session destruction | Any authenticated |

## Data Flow

### Request Lifecycle

```
1. Browser Request
      |
      v
2. hooks.server.ts  -- reads "session" cookie
      |                calls validateSession()
      |                populates event.locals.user / .session
      v
3. +layout.server.ts -- exposes user to page.data
      |
      v
4. Route Load (+page.ts / +page.server.ts)
      |                queries DB or calls API
      v
5. +page.svelte     -- renders with $props() data
      |
      v
6. Component Tree   -- Svelte 5 Runes reactivity
```

### API Data Pattern

Public pages (series listing, hero) use `+page.ts` client loads that call internal `/api/*` endpoints:

```typescript
// src/routes/(app)/series/+page.ts
export const load = async ({ fetch }) => ({
  series: fetch('/api/series?page=1').then(r => r.json())
});
```

Admin pages use `+page.server.ts` with direct DB access via `getDb()`.

## Configuration

### Environment Variables (`.env`)

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | Yes | Neon PostgreSQL connection string |
| `AUTH_SECRET` | Yes | JWT signing secret (min 32 chars) |

### TypeScript Config (`tsconfig.json`)

- `strict: true`
- `module: "NodeNext"`
- `moduleResolution: "NodeNext"`
- All server imports **must include `.js` extension**

### Tailwind v4 Theme (`src/app.css`)

- Custom colors: `coral`, `lavender`, `mint`, `plum`, `cream`
- Custom fonts: `font-display`, `font-body`, `font-thai`
- Fluid typography/spacing via `clamp()`
- Glassmorphism utilities: `.glass-card`, `.glass-card-strong`
- Animation keyframes: `float`, `orbit`, `pulse-glow`, `slideUp`, `fadeIn`
- Respects `prefers-reduced-motion`

## Security Model

- **Session**: JWT in `httpOnly` cookie, `sameSite: 'lax'`, `secure` in production
- **Passwords**: bcrypt 12 rounds
- **Token storage**: SHA-256 hash in DB (revocable)
- **CSRF**: Handled by SvelteKit form actions (default)
- **XSS**: Mitigated by Svelte's built-in escaping
- **SQL injection**: Prevented by Drizzle ORM parameterization
- **Role guard**: `admin/+layout.server.ts` redirects non-ADMIN users
